Основні команди, які використовуються в Git:


  • git config --global user.name blabla - вказати на ім'я користувача
  • git config --global user.email blabal@bla - вказати електронну пошту користувача
  • git init - створити порожній репозиторій Git або повторно ініціалізувати наявний
  • git status - показати стан робочого дерева
  • git add - додати файли в коміт
  • git commit - записати зміни до репозиторія
  • git log - показати журнали фіксації
  • git checkout - ініціалізувати та змінити реєстрацію
  • git branch - оновити віддалені посилання разом із пов’язаними об’єктами
  • git push - відправити в репозиторій зліпки проєкту
  • git pull - отримати та інтегрувати з іншим репозиторієм
  • git merge - об’єднати дві або більше історії разом
  • git clone - клонувати репозиторій в новий директорій

.gitignore - команда, яка вказує, які файли повинен ігнорувати Git

Advanced


  • git revert - безпечний спосіб відмінити операцію без втрати історії комітів
  • git reset - універсальний інструмент для відміни змін
  • git rebase - команда, яка дозволяє коміти з однієї гілки перенести на іншу
  • git cherry-pick - команда, яка бере зміни, які були внесені одним комітом і намагається повторно виконати їх в вигляді нового коміта в даній гілці
  • git stash - команда, яка дозволяє на деякий час відкласти зміни, зроблені в робочій копії, для того щоб пізніше їх застосувати
